# 星尘（Stardust）版本历史

## v3.7.2026.0201 (2026-02-01)

### 新特性
- **远程命令执行**：新增远程命令执行能力并完善安全审计
- **部署体系重构**：引入策略模式与多部署模式，支持影子模式和标准模式
- **部署资源管理**：引入部署资源与资源版本管理，支持多平台分发
  - 支持新增部署资源和上传资源版本文件
  - 发布应用时，支持选择资源包，批量下发
- **分布式文件存储**：支持文件同步，使用竞速下载，文件下载接口增加哈希相关响应头
- **节点信息增强**：大幅增强节点信息采集能力
  - 增加采集GPU板卡与显存信息
  - 增加采集时区
  - 增加采集系统负载、IOPS、网络延迟和丢包率
  - 增加采集磁盘活动时间
  - 增加采集glibc++版本和VC++运行时版本
  - 新增节点LibcVersion字段
  - Windows屏幕信息采集重构，提升兼容性与健壮性
  - 自动采集Linux分辨率与DPI
- **桥接DI容器**：桥接 ObjectContainer 与 ASP.NET Core DI 容器
- **XCode拦截器升级**：XCode拦截器体系升级

### 功能改进
- 监控调用链图增加节点字段，方便识别单应用多节点部署的情形
- 应用历史、应用命令、应用性能，增加节点字段
- 应用历史支持查询成功失败
- 应用客户端未登录时，不能执行文件操作
- 获取当前可用资源时，不包括ProjectId=0的全局资源
- 多节点执行时，避免重复计算
- 操作系统类型归类精简与识别逻辑优化
- 增加Host校验，提升服务访问安全性
- 节点管理、注册中心、监控中心，采用新的控制器基类
- 配置中心提取控制器基类

### 性能优化
- 切换为 PDH API 采集 Windows 性能计数器数据
- 优化Windows信息采集，增强GPU与性能指标获取
- 优化进程查找逻辑，提升文件路径匹配精度
- 重构服务进程查找逻辑并优化相关代码
- 优化部署策略，增强影子模式和标准模式下的文件处理逻辑，支持 zip 文件的自动识别
- 优化Linux下libstdc++版本检测逻辑与兼容性
- DPI 规范化处理
- 根据新版ApiHttpClient重写服务创建客户端逻辑

### Bug修复
- 修正令牌过期时返回ApiCode错误，应返回401却返回403，导致长时间网络不稳定的客户端无法自动重新登录
- 修正在2025-9-10错误修改rs.Code = device2.Secret，导致新应用重复注册为错误应用名
- 修正错误下发应用密钥的问题
- 修正zip文件检索逻辑
- 修正Redis事件总线没有订阅数据的问题
- 修正已用内存计算错误，需要加括号

### 数据库优化
- 在Sqlite中拆分TraceData和SampleData到多个Sqlite数据库
- 每次删除历史数据时限制100万，避免卡死
- 自动检查分表分库时，特殊支持SQLite
- 缩短资源版本和部署版本的Url字段长度

### 其他改进
- 升级核心库和魔方
- 优化百分比小数的显示
- 增加星尘发布的架构文档

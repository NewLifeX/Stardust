# 星尘（Stardust）版本历史

## v3.9.2026.0707 (2026-07-07)

### .NET 运行时自动安装与升级
- **产品版本管理**：新增产品版本、发布包、dotNet 安装包三张数据表及完整升级逻辑，支持 .NET 运行时多版本管理
- **自动同步服务**：新增 `DotNetSyncService`，自动同步官方 .NET 发布信息并支持动态调整同步周期
- **冒烟测试与回滚**：增强升级重启流程，支持冒烟测试与自动回滚，保障升级安全

### 进程管理增强
- **OOM 分值调整**：新增 OOM 分值调整逻辑，支持 Linux 下进程优先级控制（`oom_score_adj`）

### 追踪与监控优化
- **追踪解析器优化**：优化 `StarTracerResolver`，增强埋点名称解析逻辑，支持高并发场景下的线程安全处理
- **心跳检测异步缓存**：心跳网络质量检测改用异步缓存，减少阻塞，提升性能

### 命令与消息优化
- **命令下发去重优化**：服务端不再过滤重复命令，由客户端自行处理，提升命令下发可靠性
- **Remoting 依赖升级**：升级 Remoting 依赖，增强升级与链路追踪日志
- **ClientBase 优化**：首次自动更新定时器执行时间调整为 15 秒，WebSocket 连接建立后立即执行
- **[fix] 命令查询排序**：节点命令查询中的 Id 排序由升序改为降序，优先返回最新命令

### 代码质量与项目结构
- **可空类型检查**：全面开启可空引用类型检查，升级依赖并优化警告处理
- **项目结构清理**：大规模清理冗余代码与模型，移除已废弃的服务实现，优化项目结构
- **TrimSuffix 替换**：将 `TrimEnd` 替换为 `TrimSuffix`，语义更清晰
- **服务方法可空化**：调整服务方法参数为可空类型，优化命令响应机制

### 文档与设计
- **指令下发架构文档**：新增指令下发全链路架构分析文档
- **设计原型**：新增 Dashboard、节点管理、监控、部署等 HTML 设计原型

## v3.8.2026.0602 (2026-06-02)

### 链路追踪增强
- **TraceAnonymous匿名访问**：新增 `TraceAnonymous` 属性，允许未登录用户访问调用链页面，适用于公开展示或演示场景

### 事件总线升级
- **接口异步化**：`GetEventBus` 重命名为 `CreateEventBus`，`Subscribe`/`Unsubscribe` 全面支持异步，提升高并发场景下的响应能力
- **追踪链路优化**：发布时优化 TraceId 赋值逻辑，增强分布式追踪可观测性；JSON 反序列化兼容 null 值

### 依赖升级
- 升级 `NewLife.Core`、`NewLife.Redis`、`NewLife.XCode` 等核心依赖至最新版本

## v3.7.2026.0501 (2026-05-01)

### 部署增强
- **[fix] Git部署错误处理**：增强 `DeployWorker` 在 Git 克隆和拉取时的错误处理机制，提升异常情况下的部署稳定性
- **应用部署配置同步**：更新应用时自动同步关联的部署配置，优化数据库操作以避免不必要的保存

### Bug修复
- **[fix] RedisQueue反射报错**：修正通过反射使用 `RedisQueue` 时的报错问题

### 依赖升级
- 升级魔方（Cube）依赖，跟进上游更新
- 升级前端安全依赖（postcss、follow-redirects、lodash、axios 等），修复已知安全漏洞

## v3.7.2026.0303 (2026-03-03)

### 新特性
- **服务健康检查**：新增服务级健康检查能力，支持监控中心直接采集服务可用性
- **部署防火墙联动**：发布部署时支持自动放行防火墙端口，降低运维手工配置成本

### 功能改进
- 分布式文件存储增强可配置启用/禁用与多场景适配能力
- 增强 `DefaultFileStorage` 事件总线自动初始化流程，降低接入复杂度
- 优化依赖并重构 `Stardust.Extensions` 的 Http 扩展和响应头处理逻辑

### 性能优化
- 健康检查组件复用静态 `HttpClient`，减少高频检查下的连接与对象开销
- 移除全局 Http 超时配置，改为遵循单次请求超时参数，提升超时控制精度

### Bug修复
- 修复 UDP 健康检查在 `NET5.0` 下的兼容性问题
- 修复健康检查实现中的引用参数闭包问题，避免结果偏差
- 修复旧版 Redis 不支持 Stream 时事件总线不可用的问题

### 其他改进
- 升级 `XCode` 与 `Cube` 依赖，优化服务端稳定性与兼容性

## v3.7.2026.0201 (2026-02-01)

### 新特性
- **远程命令执行**：新增远程命令执行能力并完善安全审计
- **部署体系重构**：引入策略模式与多部署模式，支持影子模式和标准模式
- **部署资源管理**：引入部署资源与资源版本管理，支持多平台分发
  - 支持新增部署资源和上传资源版本文件
  - 发布应用时，支持选择资源包，批量下发
- **分布式文件存储**：支持文件同步，使用竞速下载，文件下载接口增加哈希相关响应头
- **节点信息增强**：大幅增强节点信息采集能力
  - 增加采集GPU板卡与显存信息
  - 增加采集时区
  - 增加采集系统负载、IOPS、网络延迟和丢包率
  - 增加采集磁盘活动时间
  - 增加采集glibc++版本和VC++运行时版本
  - 新增节点LibcVersion字段
  - Windows屏幕信息采集重构，提升兼容性与健壮性
  - 自动采集Linux分辨率与DPI
- **桥接DI容器**：桥接 ObjectContainer 与 ASP.NET Core DI 容器
- **XCode拦截器升级**：XCode拦截器体系升级

### 功能改进
- 监控调用链图增加节点字段，方便识别单应用多节点部署的情形
- 应用历史、应用命令、应用性能，增加节点字段
- 应用历史支持查询成功失败
- 应用客户端未登录时，不能执行文件操作

- 多节点执行时，避免重复计算
- 操作系统类型归类精简与识别逻辑优化
- 增加Host校验，提升服务访问安全性
- 节点管理、注册中心、监控中心，采用新的控制器基类
- 配置中心提取控制器基类

### 性能优化
- 切换为 PDH API 采集 Windows 性能计数器数据
- 优化Windows信息采集，增强GPU与性能指标获取
- 优化进程查找逻辑，提升文件路径匹配精度
- 重构服务进程查找逻辑并优化相关代码
- 优化部署策略，增强影子模式和标准模式下的文件处理逻辑，支持 zip 文件的自动识别
- 优化Linux下libstdc++版本检测逻辑与兼容性
- DPI 规范化处理
- 根据新版ApiHttpClient重写服务创建客户端逻辑

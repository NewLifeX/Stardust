# M8-9 网关需求文档（StarGateway）

> 版本：v1.0 | 日期：2026-07-15
> 对应模块：M8 网关管理
> 注意：本文档内 M8.1~M8.5 编码对应 StarGateway 内部模块，与主项目 M1-M12 编码体系独立

---

## 1. 背景与愿景

### 1.1 系统定位

StarGateway 是星尘分布式平台的可选组件，提供 Nginx 类反向代理能力，但**网关实例本身作为客户端应用连接到 StarServer 中心管理服务器获取配置**，无需在每个网关实例上手工配置，可与星尘注册中心协同实现服务发现与负载均衡。

| 工程 | 定位 | 部署方式 |
|------|------|----------|
| **StarGateway** | 流量网关。接收外部请求，按动态规则转发到后端应用 | 每台需要暴露公网流量的服务器部署一个实例 |
| StarAgent | 节点代理。进程守护、远程发布、监控采集 | 每台服务器必装一个实例 |
| StarServer | 服务平台后端。API / 配置 / 注册 / 发布 / 指标聚合 | 中心部署 1~N 节点 |
| Stardust (SDK) | 客户端接入库。配置拉取、注册发现、调用追踪 | 与业务应用同进程 |

### 1.2 愿景

1. **配置零手工**：部署后自动从 StarServer 拉取路由规则，无需 SSH 登录改配置
2. **与星尘平台深度协同**：自动感知星尘注册中心的服务变更，动态更新路由表
3. **应用按需启停**：与 StarAgent 协同，流量触发冷启动，空闲自动回收资源
4. **小白也能运维**：StarWeb 上可视化配置路由，无需学习 Nginx 的配置语法

## 2. 核心目标

| 编码 | 目标 | 所属层级 | 一句话描述 |
|------|------|----------|------------|
| M8.1 | 动态路由与流量转发 | 基础能力 | 支持域名/路径匹配的路由，将请求转发到指定后端，实现负载均衡 |
| M8.2 | 集中配置管理 | 控制面 | 作为 StarServer 的客户端自动获取路由规则，支持本地配置文件兜底 |
| M8.3 | StarAgent 协同 | 节点协同 | 与本地 StarAgent 通信实现应用启停、端口探测、空闲回收 |
| M8.4 | 可观测性 | 运维保障 | 访问日志、APM 链路追踪集成、运行指标暴露 |
| M8.5 | 证书与集成 | 基础设施 | 证书统一管理、多格式支持、StarServer 注册 |

## 3. 功能需求

### 3.1 M8.1 动态路由与流量转发

- **域名路由**：根据 HTTP Host 头将请求路由到不同后端，支持泛域名匹配（`*.example.com`）
- **路径路由**：根据 URL 路径前缀（`/api/*`、`/web/*`）路由到不同后端
- **请求头/查询参数路由**：根据 Header 值或 Query 参数匹配路由
- **负载均衡**：同一路由目标支持多后端节点，实现轮询/最少连接/IP Hash 等算法
- **TLS 终止**：入站 HTTPS 解密，支持 SNI 多证书，证书可从 StarServer 自动下发
- **健康检查**：对后端节点进行主动（定时心跳）和被动（失败计数）健康检查，自动摘除不健康节点
- **WebSocket 代理**：支持 HTTP Upgrade 机制的 WebSocket 协议代理，透明转发帧数据

### 3.2 M8.2 集中配置管理

- **StarServer 配置下发**：启动时连接 StarServer 获取完整路由配置，运行中通过长连接/定时轮询接收增量更新
- **本地配置文件兜底**：当 StarServer 不可达时，使用本地配置文件（JSON/YAML）继续运行
- **配置热更新**：路由配置变更后实时生效，不中断已有连接，实现零停机重载
- **本地 Admin API**：提供 HTTP API 查询运行时状态，支持拉取/手工触发配置重载

### 3.3 M8.3 StarAgent 协同

- **本地 StarAgent 通信**：通过本地 HTTP API（`127.0.0.1:{LocalPort}`）与 StarAgent 交互
- **端口探测**：对后端目标端口进行 TCP 连接探测，判断应用是否就绪
- **应用唤醒（冷启动）**：当请求到达但后端端口未监听时，通知 StarAgent 启动应用
- **空闲回收**：后端应用持续无流量超过阈值时，通知 StarAgent 停止应用以释放资源

### 3.4 M8.4 可观测性

- **访问日志**：记录请求方法、路径、状态码、响应时间、来源 IP
- **星尘 APM 集成**：转发链路的上报 TraceId/SpanId，使网关转发的请求可在星尘链路追踪中完整呈现
- **运行指标**：暴露当前活跃连接数、路由条目数、后端健康状态等指标

### 3.5 M8.5 证书与集成

- **证书统一管理**：复用星尘部署中心 SslCertificate，支持自动续期
- **多格式证书支持**：PEM / PFX / CRT+KEY 自动识别加载
- **证书热更新**：配置刷新周期自动重载证书
- **StarServer 注册**：作为 AppClient 注册到 StarServer，在线可见
- **优雅关闭**：反向顺序停止服务，每服务 10 秒超时

## 4. 🧊 暂缓清单

| 功能 | 暂缓理由 | 解冻前提 |
|------|----------|----------|
| HTTP/2 / HTTP/3 支持 | 当前 HTTP/1.1 满足 MVP 需求 | 出现必须 HTTP/2 的后端或客户端需求 |
| 请求/响应转换（插件） | 如 Kong/APISIX 的插件体系，工程量较大 | M8.1~M8.5 核心能力完成后 |
| 限流/熔断 | 需要定义配置模型和算法实现 | 负载均衡和健康检查完善后 |
| JWT/API Key 认证 | 可在后端应用层完成，网关层非必须 | 需要统一网关层认证的场景出现 |
| Prometheus 指标暴露 | 星尘 APM 已满足核心观测需求 | 用户明确要求 Prometheus 集成 |
| 配置管理 Web UI | StarWeb 上可视化配置路由规则 | StarGateway 基础能力稳定后 |

## 5. 不做什么

- **不做全局唯一网关**：StarGateway 可多实例部署，每个实例独立运行
- **不做服务网格数据面**：StarGateway 是流量网关，东西向流量由 Stardust SDK 客户端负载均衡完成
- **不做配置存储**：路由配置持久化由 StarServer 负责，StarGateway 只做配置消费方
- **不做动态证书签发**：SSL 证书管理由 StarServer 完成

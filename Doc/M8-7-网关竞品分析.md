# M8-7 网关竞品分析

> 版本：v1.0 | 日期：2026-07-15
> 对应需求：M8-7 StarGateway 竞品分析

---

## 概述

本报告对比 **StarGateway**（星尘网关）与市面上主流的反向代理 / API 网关竞品，覆盖动态配置、集中管理、服务发现集成、路由能力等关键维度。

StarGateway 定位为**连接中心管理服务器（StarServer）的客户端代理应用**，与 Envoy 的 xDS 客户端 + 控制面模式架构思路最为接近，但更轻量且与星尘平台原生集成。

## 主要竞品分析要点

| 竞品 | 核心参考价值 |
|------|-------------|
| **YARP** | `IProxyConfigProvider` 的配置热更新接口设计可直接参考 |
| **Nginx Plus** | 商业版动态配置 + 管理 API 是 StarGateway 对标方向 |
| **Kong** | Admin API 全动态配置 + CP/DP 分离架构值得借鉴 |
| **Traefik** | 自动服务发现（Docker/K8s/Consul）是最佳实践 |
| **Envoy** | xDS 协议 + LDS/RDS/CDS 是控制面标准 |

## StarGateway 的差异化优势

1. 与 StarServer 深度集成，配置自动下发零手工
2. 与 StarAgent 协同，实现应用按需启停和空闲回收
3. 与星尘注册中心协同，实现服务发现和负载均衡
4. 超轻量，单二进制文件即可运行
